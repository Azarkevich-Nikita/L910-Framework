## О проекте

**L910-Framework** — это минималистичный веб-фреймворк для Node.js, разработанный с нуля без использования готовых решений. Фреймворк предоставляет базовую функциональность для создания серверных приложений, аналогичную Express.js, но реализованную самостоятельно для глубокого понимания работы веб-серверов.

### Для чего предназначен фреймворк

Фреймворк решает следующие задачи:

- **Создание HTTP-серверов** — позволяет легко создавать и запускать веб-серверы на Node.js
- **Маршрутизация запросов** — поддерживает регистрацию обработчиков для различных HTTP-методов (GET, POST, PUT, PATCH, DELETE) и путей с параметрами
- **Обработка запросов** — предоставляет удобные объекты Request и Response для работы с данными запросов и ответов
- **Middleware** — поддерживает цепочки промежуточных обработчиков для логирования, аутентификации и других задач
- **Обработка ошибок** — встроенная система обработки ошибок, предотвращающая падение сервера

### Основные возможности

- ✅ Поддержка всех основных HTTP-методов (GET, POST, PUT, PATCH, DELETE)
- ✅ Параметризованные маршруты (например, `/users/:id`)
- ✅ Парсинг тела запроса (JSON, form-data)
- ✅ Работа с query-параметрами
- ✅ Цепочки middleware
- ✅ Обработка ошибок
- ✅ Удобные методы для формирования ответов (res.send(), res.json(), res.status())

## Как начать использовать

### Требования

Для работы фреймворка необходим Node.js.

### Установка

1. Клонируйте репозиторий
   ```sh
   git clone https://github.com/Azarkevich-Nikita/L910-Framework.git
   ```
2. Перейдите в директорию проекта
   ```sh
   cd L910-Framework
   ```
3. Установите зависимости (если есть)
   ```sh
   npm install
   ```
4. Запустите сервер
   ```sh
   npm start
   ```

Сервер будет запущен на порту 5050 (по умолчанию). Вы можете изменить порт в файле `index.js`.

## Роутинг приложения

Приложение использует модульную структуру роутинга, где каждый сервис имеет свой собственный файл с маршрутами. Все данные хранятся в JSON-файлах и обрабатываются через соответствующие эндпоинты.

### Базовые маршруты
<a id = base-routes></a>

| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/` | Возвращает "Hello World" |
| `GET` | `/users/:id` | Получить информацию о пользователе по ID |
| `POST` | `/users/:id` | Создать пользователя |

### Army (Армия)
<a id = army-routes></a>
Модуль управления военными ресурсами: солдатами и транспортными средствами.

#### Soldiers (Солдаты)

| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/soldiers` | Получить список всех солдат |
| `GET` | `/soldiers/:id` | Получить информацию о солдате по ID |
| `POST` | `/soldiers` | Создать нового солдата |
| `PUT` | `/soldiers/:id` | Полностью обновить данные солдата |
| `PATCH` | `/soldiers/:id` | Частично обновить данные солдата |
| `DELETE` | `/soldiers/:id` | Удалить солдата |

**Поля солдата:**
- `id` (автоматически генерируется)
- `name` - имя
- `rank` - звание
- `age` - возраст
- `isActive` - статус активности
- `enlistedAt` - дата призыва
- `skills` - навыки

#### Vehicles (Транспортные средства)

| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/vehicles` | Получить список всех транспортных средств |
| `GET` | `/vehicles/:id` | Получить информацию о транспортном средстве по ID |
| `POST` | `/vehicles` | Создать новое транспортное средство |
| `PUT` | `/vehicles/:id` | Полностью обновить данные транспортного средства |
| `PATCH` | `/vehicles/:id` | Частично обновить данные транспортного средства |
| `DELETE` | `/vehicles/:id` | Удалить транспортное средство |

**Поля транспортного средства:**
- `id` (автоматически генерируется)
- `model` - модель
- `type` - тип
- `crew` - экипаж
- `isOperational` - работоспособность
- `lastService` - последнее обслуживание
- `equipment` - оборудование

### Примеры использования

#### Получить список всех солдат
```bash
GET http://localhost:5050/soldiers
```

#### Создать нового солдата
```bash
POST http://localhost:5050/soldiers
Content-Type: application/json

{
  "name": "Иван Иванов",
  "rank": "Сержант",
  "age": 25,
  "isActive": true,
  "enlistedAt": "2020-01-15",
  "skills": ["Стрельба", "Тактика"]
}
```

#### Обновить данные актера (частично)
```bash
PATCH http://localhost:5050/actors/1234567890
Content-Type: application/json

{
  "age": 30,
  "isMovieActor": true
}
```

#### Удалить посетителя
```bash
DELETE http://localhost:5050/visitors/1234567890
```

### Обработка ошибок

Все эндпоинты возвращают соответствующие HTTP-коды статуса:
- `200` - Успешный запрос
- `201` - Ресурс успешно создан
- `404` - Ресурс не найден
- `500` - Внутренняя ошибка сервера

При ошибке возвращается JSON-объект с описанием ошибки:
```json
{
  "error": "Soldier not found"
}
```
